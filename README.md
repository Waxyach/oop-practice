### 📝 Виконані завдання

#### **Завдання 1: Розрахунок струмів у паралельних опорах**

Розробка системи для обчислення струму, що проходить крізь чотири паралельно підключені опори.

* **Реалізовано:** серіалізація об'єктів, використання `transient` полів для оптимізації.
* **Тестування:** повне покриття логіки за допомогою JUnit.

#### **Завдання 2: Спадкування, інтерфейси та колекції**

Розширення функціональності проєкту шляхом впровадження об'єктно-орієнтованих патернів та роботи з колекціями.

* **Патерн Factory Method:** Реалізовано ієрархію View та Viewable для абстракції створення об'єктів відображення. 

* **Колекції:** Використано ArrayList для зберігання та маніпуляції результатами обчислень. 

* **Функціональні інтерфейси:** Використано лямбда-вирази для лаконічної ініціалізації фабрики.

* **Тестування:** Перевірка цілісності всієї колекції при збереженні та відновленні даних.

#### **Завдання 3: Поліморфізм та форматований вивід**

Розширення системи можливостями текстового представлення даних у вигляді таблиць.

* **Поліморфізм:** Реалізовано перевизначення методів (overriding) viewHeader, viewBody та viewFooter для зміни формату виводу під час виконання. 

* **Перевантаження:** Додано декілька версій методів init та конструкторів для гнучкого налаштування параметрів таблиці (наприклад, ширини). 

* **Форматований вивід:** Використано класи java.util.Formatter та printf для створення рівних колонок таблиці в консолі. 

* **Тестування:** Модульні тести для перевірки розрахунків та коректності відновлення даних колекції.

#### **Завдання 4: Обробка колекцій. Шаблони Singleton та Command**

Розробка розширюваної системи керування командами та алгоритмів обробки колекцій[cite: 384, 389].

* **Шаблон Singleton:** Клас `Application` реалізовано як Одинак для централізованого доступу до ресурсів програми.
* **Шаблон Command:** Кожна дія користувача (View, Generate, Save, Restore) винесена в окремий об'єкт-команду, що дозволило відокремити інтерфейс від бізнес-логіки.
* **Макрокоманда та Undo:** Клас `Menu` реалізовано як контейнер команд із підтримкою скасування (`undo`) останньої операції].
* **Обробка даних:** Реалізовано алгоритм групової обробки колекції (масштабування параметрів усіх об'єктів одночасно).
* **Тестування:** Перевірка методу `execute()` для команд зміни даних та цілісності стану після скасування операцій.
